C++中带括号的new。。（placement new）
2010-06-16 20:35
(hplonline)2010.6.16

》》起因

之前有同学做个在线笔试，碰到了这个题：

#include <iostream>
using std::cout;
using namespace std ;

class P
{
public:
virtual void print()
{
cout << "P";
}
};

class Q: public P
{
public:
virtual void print()
{
cout << "Q";
}
};

int main()
{
P * p = new P;
Q * q = static_cast <Q *> (p);
q->print();
delete p;
cout << endl;

q = new Q;
p = q;
q->print();
p->print();
cout << endl;

p = new (q) P;
q->print();
p->print();
cout << endl;
p->~P();
delete q;

return 0;
}

题目当然是让你判断会出什么结果。

对于前两个，应该说比较好搞定。
第一个，static_cast，
在这里和平时用的强制类型转换是一样的。
当存在多重继承的时候，会有一些猫腻，
详情见笔记：C++类型转换(static_cast,reinterpret_cast)
第二个，就是一般的虚函数的问题，
创建的对象是什么那就调用谁的函数。

比较诡异的是第三个，
new(q)P这种用法还真是第一次见到。
（深刻表示土鳖。。以前就是没看到过。。
并且这个东西没法构造关键词在网上搜。。）

》》new的几种常用写法

new XXX
这种必须要配合delete 来使用。

new XXX[]
这种必须要配合delete[]来使用。

另外可以用括号来初始化创建的对象，
括号的位置相当于是构造函数的位置。
比如：new int(3) ;

今天遇到的这种形式为：
new(??) XXX

》》观察目标代码的结论

从目标代码来看，p=new(q)P做了以下事情：
1。直接让p的值等于q
2。以p这个地址为this指针，调用了P的构造函数。

如果觉得看汇编太麻烦，可以用C++写个更简洁点的实验：

#include <iostream>
using namespace std;

class cls1{
public:
int i ;
cls1(){
i = 1 ;
}
void print(){
cout<<i<<endl ;
}
} ;

class cls2{
public:
int i1 , i2 ;
cls2(){
i1 = 2 ;
i2 = 2 ;
}
void print(){
cout<<i1<<','<<i2<<endl ;
}
} ;

int main(){
int *p ;
cls1 *p1 = new cls1 ;
p1->print() ;
cls2 *p2 = new(p1) cls2 ;
p2->print() ;
p1->print() ;
return 0 ;
}

输出：

1
2,2
2

这个实验也可以证实两个指针的值相同，
并且后者的构造函数已经被调用了。

》》空间分配的问题

这里，我们又遇到了另一个很矛盾的问题：
new(??) XX是进行简单的指针赋值呢，还是重新分配空间？

如果不重新分配空间的话，
新类和旧类所占空间不一定是相同的，
这样，有的数据成员可能就已经存到了无效区域。

还好，new出来的东西会有一个cookie，
在指针前面某地，是存储得有空间大小的。
根据这个，我们可以设计实验来观察。

实验：

int main(){
int *p1 , *p2 ;
p1 = new int[12] ;
printf("%d\n" , p1[-4]) ;

p2 = new(p1) int[13] ;
printf("%d\n" , p2[-4]) ;

p2 = new int[13] ;
printf("%d\n" , p2[-4]) ;
return 0 ;
}

输出：

48
48
52

可以看到new(p1) int[13] ;出来的cookie，
标注空间大小为48，和原来一样。
new int[13] ;出来的coockie，
则是和新申请的一样，52。

这说明new(??)XX的形式并没有重新分配空间。
在上面这个实验中，使用的是基础类型int，
这种类型具有trivial的dtor和ctor，
实验中，delete p1或者p2都是正常的。

但下面的实验将展示出它的一个副作用：

#include <iostream>
using namespace std;

class cls1{
public:
int i ;
cls1(){
i = 1 ;
}
void print(){
cout<<i<<endl ;
}
} ;

class cls2{
public:
int i1 , i2 ;
cls2(){
i1 = 2 ;
i2 = 2 ;
}
void print(){
cout<<i1<<','<<i2<<endl ;
}
} ;

int main(){
int *p ;
cls1 *p1 = new cls1 ;
p = (int*)p1 ;
cout<<p[0]<<','<<p[1]<<endl ;
cls2 *p2 = new(p1) cls2 ;
p = (int*)p2 ;
cout<<p[0]<<','<<p[1]<<endl ;

//delete p1 ;
//delete p2 ;
//both will fail here!!

return 0 ;
}

在以上语句执行后，
不管delete哪个指针，
最后都会出现错误！

这是由于cls2 *p2 = new(p1) cls2 ;没有重新分配空间，
也就是p2指向的地方只有一个int的位置，
但是cls2的构造函数很明显地向两个地方写入了数据，
后者已经在有效空间范围外了。

》》总结

由于没有重新分配空间，
所以new(??)XXX这样的用法是非常危险的。
当新对象所占空间小于原分配空间的时候，不会出问题。
但当新对象所占空间比之大的时候，可能造成对非法区域的改写。

那是不是就要完全放弃这种写法呢？？

显然不是！
第一，就像上面分析的，如果能保证新对象的空间更小，则无所谓。
C的哲学不是让我们放弃一切危险的动作，
而是让我们深入了解每个动作，让危险可控。

第二，这给我们手动调用构造函数提供了一个方法！
其实这个问题也困扰了我很久，如今却无心插柳地发现了玄机。
从OD中对目标代码的分析，和几个实验，
可以看到，这个写法仅仅是在原指针位置调用了一下新对象的构造函数。

我们知道，一般情况下，我们是无法手动调用构造函数的，
ctor都是在new或者栈变量初始化的时候，由编译器调用的。
而现在，我们可以通过这种写法来调用ctor，如下：

int main(){
cls1 *p = new cls1() ;
p->print() ;
p->i = 2 ;
p->print() ;
p = new(p) cls1() ;
p->print() ;
return 0 ;
}

输出：

1
2
1

其实。。。调用构造函数还是可以这么写的：
p->cls1::cls1() ;

在VC6里面是过了，但应该是不标准的用法。

又查了些东西，知道了本篇一直讨论的形式原来叫：placement new

感觉这个东西就是和手动析构联合起来用的：
先手动调用一个对象的dtor，然后使用placement new调用ctor。
如果不是同一类对象，使用placement new有潜在的空间分配问题。
或者是程序员能从某处拿到非new方法（比如malloc分配的空间），
于是使用placement new来初始化这段空间。
