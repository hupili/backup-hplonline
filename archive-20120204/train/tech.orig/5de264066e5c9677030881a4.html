Apache+Asp.Net的基本配置记录
2010-05-06 22:11
(hplonline)2010.5.6

一。下载

Apache下载：http://httpd.apache.org/download.cgi
（我下的是2.2.15）

Mod_AspdotNet下载：http://sourceforge.net/projects/mod-aspdotnet/
（官方表示开发人员不够，停止对这个模块的开发，
所以到原作者SourceForge的页面下载）

二。安装

其实没啥好说的，
不过在VISTA下面折腾UAC是一件囧事，
反正害得我第一次安是失败的。

按照默认的参数安了，
访问http://localhost/，
应该可以看到 It works的字样，就OK了。

三。基本配置

我刚一装好，到处想找诸如IIS那样的界面，
比如在什么地方填下目录，在什么地方配置下别名等。
结果显然是没找到。
唯一的一个GUI就是Monitor，
其功能也就是开启/停止/重启服务器这些。。

其配置全部都浓缩在httpd.conf这个文件里了，
默认的路径应该是：
C:\Program Files\Apache Software Foundation\Apache2.2\conf\httpd.conf
用记事本打开即可，
vista同样要注意UAC的问题，
要不修改了保存不了。。

修改几个地方即可呈现基本的页面。
设网站放在 d:\web，默认页面是index.htm。
紫色是原内容，可以先ctrl+f到，然后修改。
每修改一处，可以用monitor restart一下server看效果。

DocumentRoot "C:/Program Files/Apache Software Foundation/Apache2.2/htdocs"
DocumentRoot "d:/web"

<Directory "C:/Program Files/Apache Software Foundation/Apache2.2/htdocs">
<Directory "d:/web">

在第二处的上面有句话：
# This should be changed to whatever you set DocumentRoot to.
所以很显然地把两个路径改成一样。。

有了这两个之后，输入完整路径，应该可以访问d:\web下的东西了。

紧接着第二处的地方：
Options Indexes FollowSymLinks
Options -Indexes FollowSymLinks

这样，服务器不会列出目录。

再往下走：
DirectoryIndex index.html
DirectoryIndex index.htm index.html

当访问目录的时候，自动导向的页面填在这里。

最后随便找个地方添句话：
AddDefaultCharset GB2312

否则，直接访问localhost而定向到的index.htm可能是乱码。

四。Asp.Net

译自文章。

在httpd.conf后面添加内容：
-------------

#asp.net
LoadModule aspdotnet_module "modules/mod_aspdotnet.so"

AddHandler asp.net asax ascx ashx asmx aspx axd config cs csproj licx rem resources resx soap vb vbproj vsdisco webinfo

<IfModule mod_aspdotnet.cpp> 
  # Mount the ASP.NET /asp application
  AspNetMount /SampleASP "c:/SampleASP"
#/SampleASP is the alias name for asp.net to execute
#"c:/SampleASP" is the actual execution of files/folders  in that location

  # Map all requests for /asp to the application files
  Alias /SampleASP "c:/SampleASP"
#maps /SampleASP request to "c:/SampleASP"
#now to get to the /SampleASP type http://localhost/SampleASP
#It'll redirect http://localhost/SampleASP to "c:/SampleASP"

  # Allow asp.net scripts to be executed in the /SampleASP example
  <Directory "c:/SampleASP">
Options FollowSymlinks ExecCGI
Order allow,deny
Allow from all
DirectoryIndex index.htm index.aspx
#default the index page to .htm and .aspx
  </Directory>

  # For all virtual ASP.NET webs, we need the aspnet_client files
# to serve the client-side helper scripts.
  AliasMatch /aspnet_client/system_web/(\d+)_(\d+)_(\d+)_(\d+)/(.*) "C:/Windows/Microsoft.NET/Framework/v$1.$2.$3/ASP.NETClientFiles/$4"
<Directory "C:/Windows/Microsoft.NET/Framework/v*/ASP.NETClientFiles">
Options FollowSymlinks
Order allow,deny
Allow from all
</Directory>
</IfModule>
#asp.net

------------

在C:\SampASP\ 下面创建index.aspx

------------

<%@ Page Language="VB" %>
<html>
<head>
<link rel="stylesheet"href="intro.css">
</head> 
<body> 
<center> 
<form action="index.aspx" method="post"> 
<h3> Name: <input id="Name" type=text> 
Category:  <select id="Category" size=1> 
<option>One</option>
<option>Two</option>
<option>Three</option>
</select> 
</h3> 
<input type=submit value="Lookup"> 
<p> 
<% Dim I As Integer
For I = 0 to 7 %>
<font size="<%=I%>"> Sample ASP.NET TEST</font> <br>
<% Next %> 
</form> 
</center> 
</body>
</html>

---------------

重启服务器，访问地址：http://localhost/SampleASP/index.aspx




这个路径居然是区分大小写的。。

五。评论：

一开始摸不着头脑，
一旦摸着了感觉很方便。
在vista下折腾了很久的iis7，
总是遇到各种诡异的问题。

纯文本的配置方式很不错，
不管是修改起来还是备份配置都是很方便。

服务器主程序加ASP.NET模块总共不超过6M。
