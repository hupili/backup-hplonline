可变参数（C,C++,stdarg.h,全可变参数实现)
2010-01-20 14:09

(hplonline)2010.1.20

今天看到湖哥在校内上讨论技术问题。
隐约对三个点有点印象，于是又细看了一遍。
简单记几个例子，以后翻来瞟一下就知道啥意思了。

以下都是VC6的结论。
C表示后缀以.c存储代码，
C++表示后缀以.cpp存储代码。
IDE根据后缀来决定调用哪种编译器。


一。C和C++里的可变参数：

#include <stdio.h>

void print(int n , ...){
    int i ;
    int *p = &n ;
    p ++ ;
    for ( i = 0 ; i < n ; i ++ ){
        printf("%d\n" , *p) ;
        p ++ ;
    }
}

int main(){
    print(3 , 1 , 2 , 3) ;
    return 0 ;
}

这里至少有一个参数，
主要是从这个参数，可以推知后面参数的地址。

二。C++支持的写法：

void print(...)
这样所有参数都是可有可无的了，
可以通过编译，
至于怎么索引参数后面再说。

C直接三个点的话，不让编译。

三。C支持的写法：

void print(int n){
    int i ;
    int *p = &n ;
    p ++ ;
    for ( i = 0 ; i < n ; i ++ ){
        printf("%d\n" , *p) ;
        p ++ ;
    }
}

int main(){
    print(3 , 1 , 2 , 3) ;
    return 0 ;
}

和前面相比，就是把后面三个点给去掉了。
C是允许函数没有定义就使用的，
他会假装是个返回int的函数。
而调用方式和现有定义不一致的时候，
也是会通过的，比如下面这样都行：
    print((float)(3) , 1 , 2 , 3) ;

不过上面那段程序编译后，
编译器会给出警告：
warning C4020: 'print' : too many actual parameters

而实际上，C可以把所有东西都省掉：
void print()
至于这样该怎么获得参数，也放在后面说。

四。使用<stdarg.h>来标准化可变参数的操作：

#include <stdio.h>
//#include <varargs.h>
#include <stdarg.h>

void print(int n , ...){
    va_list l ;
    va_start(l , n) ;
    while ( n -- ){
        int num = va_arg(l , int) ;
        printf("%d\n" , num) ;
    }
    va_end(l) ;
}

int main(){
    print(5 , 1 , 2 , 3 , 4 , 5) ;
    return 0 ;
}

va_list是参数列表定义。
va_start（参数列表，最后一个固定参数）
--在开始的时候调用
va_arg（参数列表，类型）
--将参数列表移动向下一个参数位置，并返回“类型”所指定的参数
va_end（参数列表）
--在结束的时候调用

其中，varargs.h里面也有几个名字相同的宏。
不过他的va_start没第二个参数，感觉很奇怪，
暂时没有时间深入研究，以后再看。

五。stdarg中宏的定义：

这个头文件中有若干条件编译，
跟我平台上对应的是：

#define _INTSIZEOF(n)   ( (sizeof(n) + sizeof(int) - 1) & ~(sizeof(int) - 1) )
#define va_start(ap,v) ( ap = (va_list)&v + _INTSIZEOF(v) )
#define va_arg(ap,t)    ( *(t *)((ap += _INTSIZEOF(t)) - _INTSIZEOF(t)) )
#define va_end(ap)      ( ap = (va_list)0 )

_INTSIZEOF这个宏用来取得某个类型按照int对齐后的尺寸。
由于在各个平台上int的定义有所不同，
另外一般对栈的push和pop操作都是以CPU字长为单位的，
所以需要这样一个宏，来取得该类型实际所占的空间。
仔细体会一下他是怎么办到的，其实很有意思。

n按照m向上取整的一般写法为：
(n+m-1) / m * n
而当m是二的方冪的时候，可以直接把低位清零而得到，
就形成了这个宏的写法。其实这样是等效的：
-sizeof(int)
“-”在补码中的定义本身就是取反加1.

va_start(ap,v) ( ap = (va_list)&v + _INTSIZEOF(v) )
v是最后一个固定参数，
那么将参数列表指针ap赋为他的下一个位置即可。

va_arg(ap,t)    ( *(t *)((ap += _INTSIZEOF(t)) - _INTSIZEOF(t)) )
这个比较普通，把指针向后移了再返回来取这个出类型为t的一个参数。

我一开始有个问题就是，
为什么不一开始就让表指针指向参数前一个单元，
每次取参数的时候，先移动指针，再返回那个位置上的值。
类似于这样：
*(t*)(ap += ???)

这里的毛病就在于，
移动指针需要知道前一个元素的类型，
这样才能确定需要动多少；
而返回参数需要知道当前元素的类型。
于是一个可能的宏会这样定义：
va_arg(ap , prev_t , cur_t)

这样写起来显然不是很爽。
程序员需要记忆前一个和当前元素的类型。
后面将实现消耗一定存储空间，
让电脑来记忆的写法。

六。全可变参数的写法（C内联汇编)：

上面的stdarg给出的解决方案，
从几个宏就可以明显地看出，
必须得有至少一个固定参数，
否则va_start就没法调用。

又如，我想实现这样的功能，
把传进去的参数都累加起来，
参数以0表示结束。
（类似C字符串的思想）
因为当数很多的时候，
我不想去数我要加多少个数。
或者当加数个数改变的时候，
我直接插到中间就行了，
只要保证最后一个0。

这两点都刺激我们寻找一种全可变参数的写法。
在前面的例子中，已经手动遍历过参数列表了，
但这里的问题在于如何得到第一个参数的位置：

#include <stdio.h>
int add(){
    int sum = 0 ;

    int *p ;
    __asm{
        lea eax , [ebp + 08h] ;
        mov p , eax ;
    }

    while ( *p ){
        sum += *p ;
        p ++ ;
    }

    return sum ;
}

int main(){
    printf("%d" , add(7 , 2 , 1 , 3 , 4 , 0)) ;
    return 0 ;
}

这段代码显然不是各种环境通用的。
只要知道了VC编译出来的函数怎么布局内存，
自然就能想到该怎么办。
栈中的结构一般来说是
[ebp + ???h]--参数n
[ebp + ???h]--参数n-1
[ebp + ???h]--。。。
[ebp + 008h]--参数0
[ebp + 004h]--返回地址
[ebp + 000h]--ebp
[ebp - 004h]--局部变量0
[ebp - 008h]--局部变量1
[ebp + ???h]--。。。
[ebp + ???h]--局部变量n
[ebp + ???h]--保护寄存器

（有时候好像还会保存一个框架下界的指针，
与保存的ebp对应，便于退出函数时进行检查。
这时候的参数0位置是[ebp + 0ch]，
具体情况看就行了。）

七。全可变参数（纯C)：

有了上面对栈结构的认识之后，
不用内联汇编，也可以找到这个地址。
只是这个时候需要借助至少一个局部变量进行界定。

int add(){
    int sum = 0 ;
   
    int *p ;
    p = &sum + 3 ;

    while ( *p ){
        sum += *p ;
        p ++ ;
    }

    return sum ;
}

八。stdarg宏的变换写法：

int __prev_length ;
#define _INTSIZEOF(n)   ( (sizeof(n) + sizeof(int) - 1) & ~(sizeof(int) - 1) )
#define va_start(ap,v) ( ap = (va_list)&v + _INTSIZEOF(v) ) , __prev_length = 0
#define va_arg(ap,t)    ((ap += __prev_length) , __prev_length = _INTSIZEOF(t) , *(t *)ap)
#define va_cur(ap,t)    (*(t *)ap)
#define va_end(ap)      ( ap = (va_list)0 )

首先开一个变量记录上一个参数的尺寸。
在初始化的时候给这个参数置零。
每次取参数可以先移动，
再保存当前变量的尺寸，
最后再返回当前参数。

以前有很长段时间觉得逗号表达式没什么用。
因为一般用逗号表达式的地方，
直接用分号都可以拆开成好几个表达式。
但是在这种类似的单行宏里面就有用了，
语句可以依次执行，并且最后一个表示为整个逗号表达式的返回值。
当va_arg(ap,t)单独成一句话的时候，
用分号和逗号分割倒是问题不大。
但是可能出现 if ( va_arg(ap , t) ){...}
如果是分号的话，就会把客户端的语句给切开，
最后导致编译错误。
