1-n的数组成的huffman树的权重
2009-10-29 16:27

(hplonline)2009.10.29

在某群上听到这个题，
正好没啥事干，找了下规律，弄了个结论出来。

首先，关于huffman树的定义这里不重复了。
然后权重的意思就是每个节点的值乘上路径长度的总合。

先企图直接推点东西出来，找不到下手的地方。
故干脆把前面一些n值对应的权重打出来，然后找规律。
用之前写过的程序，比较方便：
acl(1:n,huffman(1:n))

于是在matlab里面用polyfit试了下，
发现各种低阶多项式都是有残差的，
也就是说没有完全满足这个规律的。

在一个群友的提示下，前后作差试试。
出来的数列b有点那个意思了，像个等差数列。
再次差分，能够得到公差k。

看一下各个k值对应的数据个数：
4，3
5，6
6，12
7，24
。。。

看来这个就是要找的规律了。
对于每一段
b = k * (m - 1) + b1 ;
a = (m - 1) * b1 + k * ( m - 2 ) * ( m - 1 ) / 2 + a1 ;
m即为从段首开始的计数。

设L为段长，那么从上一个段的首项推下一个段的首项，
取的m应该为L+1。

从第一段开始递推，即可得到答案。
因为段长是每次加倍的，
所以能在n的对数级解决问题。

演示程序如下：

n = 90 ;%要求的输入
n = n - 2 ;
a1 = 3 ;
b1 = 6 ;
L = 3 ;
k = 4 ;
while L <= n
    m = L + 1 ;
    b = k * (m - 1) ;
    a = (m - 1) * b1 + k * ( m - 2 ) * ( m - 1 ) / 2 ;
    n = n - L ;
    L = L * 2 ;
    k = k + 1 ;
    a1 = a + a1 ;
    b1 = b + b1 ;
   
end

n = n + 1 ;
m = n ;
a = (m - 1) * b1 + k * ( m - 2 ) * ( m - 1 ) / 2 ;
a1 = a + a1 ;

result = a1

由于是演示，就没作输入检验了。
因为这个关系是从2开始的，
所以对输入1，特殊输出为0即可。
