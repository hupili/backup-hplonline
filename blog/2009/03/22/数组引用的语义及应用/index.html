数组引用的语义及应用

Mar 22nd, 2009

(hplonline)2009.3.22

此篇的实验在DEV C++完成，因为有些特性VC编译并不支持。

引用，也一直忽略了很久。因为从最底层来看，大部分引用都是用指针来实现的。
于是，很多书也给出了这样的结论：
引用就是指针，或者说是一种让编译器自动脱指针运算的方式。

对于一般的引用，通过一个例子来验证以上说法：
int swap(int &a , int &b){
    int t = a ;
    a = b ;
    b = t ;
}

那么把所有引用的声明位置替换成指针声明。把所有引用使用的位置替换成脱指针操作。
就可以得到一个等价的程序

int swap(int a , int b){
    int t = a ;
    a = b ;
    b = t ;
}
（注：这两段代码是为了表意直接在页面上写的）

这样的一段话实现同样的功能，这让我们更加坚定以上结论。
他在大多数时候描述的就是事实。而后面给出的替换法则也是通常成立的。

正如上回偶然看到的数组和指针的差异一样。
今天突然发现了数组引用的另外的特性。

先上一段代码吧:

void f(int a[5]){
     cout<<sizeof(a)<<endl;    
}

void g(int *a){
     cout<<sizeof(a)<<endl;    
}

void h(int (&a)[5]){
     cout<<sizeof(a)<<endl;
}

int main(){
    int a[5];
    f(a);
    g(a);
    h(a);
    system(“pause”);
    return 0 ;
}

输出:
4
4
20

前两个说明了一句，我搞忘是哪里说的了：
作为函数的参数，指针和数组是一样的。

也就是第一个数组虽然给了长度5，但是，作为函数参数，处理为一个指针，丢失了长度信息。

而最后一个，就是土鳖我今天发现的新大陆。
数组的引用，携带了有关数组长度的信息。

这里，引用&这种修饰符是针对变量的。所以要用括号结合起来。

不过这里既然是引用，那么引用和被引用双方一定得是相同的东西了。
所以如果把a[5];改成a[6];这下编译器就报错了。

那么，同时在网上看到的一段很通用的模板代码：

template<typename T, size_t N>
void f(T (&a)[N])
{
   cout<<sizeof(a)<<endl;
}

int main(){
    int a[6];
    char c[3];
    f(a);
    f(c);
    system(“pause”);
    return 0 ;
}

输出：
24
3

可以适应不同类型和大小的数组。

可见。。以前对引用的认识不完全。
那么今后会得到怎样的认识呢。。期待。
