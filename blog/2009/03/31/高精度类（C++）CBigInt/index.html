高精度类（C++）CBigInt

Mar 31st, 2009

(hplonline)2009.3.31

高精度拿来干什么地球人都很清楚了。
以前写就只顾着能出结果就行。
后来发现要写个稍微通用点的还是挺麻烦。

我这个显然就是不太通用的。

有些细节需要说一下。

1.只能应对数据不断增加的正整数。

首先只考虑了正整数的加法和乘法。
然后通过阅读相关实现就知道。
如果data[l]之后的数据不全为0，则会出差错。
其实我接触到的很多代码都有这毛病，只是没人说出来罢了。

不说的原因也很好理解。偶尔在OJ上刷题，用不着考虑那么全面。
我们做高精度类题遇到的数据一般都是正整数。
给的递推公式一般也只包含乘法和加法。
自然就不用考虑那么多了。

2.mul函数中开的tmp[]，在某些地方得new出来才行。

像TOJ，堆栈只给了8k的样子。一调这函数就爆掉了。

而这里直接开在函数里面，也是为了适应小数据时对速度的需求。
（new这个东西总得来说很慢。。delete[] 更慢）

operator+(和operator(就已经是new出临时变量了。
因为他们用得很少，或者说可以不用。
刷题的时候完全可以把式子转换成 +=和=系列。
这两个系列省去了无聊的中间赋值，更快。

3.输出

输出没有重载了。cout本身就一蜗牛。。给了个printf版和putchar版的。
G++下，getchar和putchar速度优势比较明显。

4.扩展

这个类全部是一个字节对应一个10进制位的。
要扩展成多位的也容易。输出要注意就是了。

代码：

const int MAXL = 30000 ;

class CBigInt{
public:
    char data[MAXL] ;
    int l ;
    CBigInt() ;
    CBigInt(int i) ;
    CBigInt(CBigInt &bi);
    CBigInt& add(CBigInt &bi) ;
    CBigInt& mul(CBigInt &bi) ;
    CBigInt& operator+(CBigInt &bi) ;
    CBigInt& operator(CBigInt &bi) ;
    CBigInt& operator+=(CBigInt &bi) ;
    CBigInt& operator=(CBigInt &bi) ;
    CBigInt& operator=(CBigInt &bi) ;
    void init(int i);
    void clear() ;
    void output() ;
};

void CBigInt::init(int i){
    l = 0 ;
    memset(data,0,sizeof(data)) ;
    do{
        data[l] = i % 10 ;
        i /= 10 ;
        l ++ ;
    }while(i);
}

void CBigInt::clear(){
    l = 1 ;
    memset(data,0,sizeof(data)) ;
}

CBigInt::CBigInt(){
    clear();
}

CBigInt::CBigInt(int i){
    init(i);
}

CBigInt::CBigInt(CBigInt &bi){
    memset(data,0,sizeof(data));
    int i ;
    l = bi.l ;
    for ( i = 0 ; i < l ; i ++ ) data[i] = bi.data[i] ;
}

CBigInt& CBigInt::add(CBigInt &bi){
    int i = 0 ;
    int p = 0 ;
    while ( i < l || i < bi.l ){
        p += data[i] + bi.data[i] ;
        data[i] = p % 10 ;
        p /= 10 ;
        i ++ ;
    }
    if ( p ) {
        data[i] = p ;
        i ++ ;
    }
    l = i ;
    return this ;
}

CBigInt& CBigInt::mul(CBigInt &bi){
    int tmp[MAXL] ;
    int i , j , p ;
    memset(tmp,0,sizeof(tmp)) ;
    for ( i = 0 ; i < l ; i ++ ){
        for ( j = 0 ; j < bi.l ;j ++ ){
            tmp[i + j] += data[i] * bi.data[j] ;
        }
    }
    j = l + bi.l + 1;
    p = 0 ;
    for ( i = 0 ; i < j ; i ++ ){
        p += tmp[i] ;
        data[i] = p % 10 ;
        p /= 10 ;
    }
    if ( p ){
        data[i] = p ;
        i ++ ;
    }
    while ( !data[i] ) i – ;
    l = i + 1 ;
    return this ;
}

CBigInt& CBigInt::operator+(CBigInt &bi){
    CBigInt p = new CBigInt(this) ;
    return p->add(bi) ;
}

CBigInt& CBigInt::operator(CBigInt &bi){
    CBigInt p = new CBigInt(this) ;
    return p->mul(bi);
}

CBigInt& CBigInt::operator+=(CBigInt &bi){
    add(bi);
    return this ;
}

CBigInt& CBigInt::operator=(CBigInt &bi){
    mul(bi);
    return this ;
}

CBigInt& CBigInt::operator=(CBigInt &bi){
    int i;
    memset(data,0,sizeof(data));
    l = bi.l ;
    for ( i = 0 ; i < l ; i ++ ) data[i] = bi.data[i] ;
    return *this ;
}

void CBigInt::output(){
    int i;
    for ( i = l - 1 ; i >= 0 ; i – ) printf(“%d”,data[i]) ;

}
