诡异的编译期求值

May 10th, 2009

(hplonline)2009.5.10

int main()
{
    int val ;
    val = 2147483647 > -2147483648;
    cout<<val<<endl;
    val = 2147483647 > -2147483647 - 1 ;
    cout<<val<<endl;
    return 0;
}

结果：

0
1

就是这样。。。试了VC6,DEV结果都是。

而val的值是在编译期就算好了的。比如：
00401798   mov     dword ptr [ebp-4], 0

怀疑编译器默认是用无符号比较的。

试一下转换：

val = 2147483647 > (signed)-2147483648;

出来的就是1了。。。

而对后面一个式子的正确结果的解释是，
-2147483647 - 1
作为一个符号数的表达式，按照符号数运算。
同时做 > 的时候也按照符号数比较，所以结果正确。

看来使用常量const是一个好习惯。
可以尽量得到期望的值。

比如：
const int a = 2147483647 ;
const int b = -2147483648 ;

…

val = a > b ;

这样通过const给相关的数据确定了是否带符号。
