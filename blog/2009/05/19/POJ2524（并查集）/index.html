AC1，普通
AC2，对初始化进行优化，少memset了一些地方，快了10多ms
AC3，加入启发式合并，没发现变快
AC4，加入外挂，刷到47ms，已经很快了
AC5，又去掉启发式合并，32ms，成功刷版

于是得出以下结论：

1.
memset少一点可以提高效率，
（估计32ms和后面47ms的就是这里有点差异）
但要小心，是以可能出错为代价的。。不要写囧。（我曾经在这个问题上囧过多次了）

2.
启发式合并是一个理论上很好的东西，但是可能由于加入过多的if，
以及路径压缩本身就很快的原因，导致没有啥优势。

3.
外挂是个很强的东西。

并查集的代码：

以后有的哥们是搜并查集进来的。。
要是看到我一堆废话肯定会起火的。。
于是给出一套，写得不好看也不丑，用起来手感很好。

      const int MAXN = 50010 ;

      int f[MAXN] ;
      //int r[MAXN] ;

      void init_set(int n){
          memset(f + 1 , 0 , sizeof(f[0]) * n) ;
      //    memset(r + 1 , 0 , sizeof(f[0]) * n) ;
      }

      int find_set(int x){
          int fa = x , t ;
          while ( f[fa] ) fa = f[fa] ;
          while ( f[x] ){
              t = x ;
              x = f[x] ;
              f[t] = fa ;
          }
          return fa ;
      }

      int union_set(int a , int b){
          a = find_set(a) ;
          b = find_set(b) ;
          if ( a == b ){
              return 0 ;
          }else{
              f[a] = b ;
      //        if ( r[a] > r[b] ) {
      //            f[b] = a ;   
      //        }else{
      //            f[a] = b ;
      //            if ( r[a] == r[b] ){
      //                r[b] ++ ;
      //            }
      //        }
              return 1 ;
          }
      }

其中注释掉的就是启发式合并的地方，
非要用启发式合并的话，就把红色的注释掉，把注释掉的变有效
点的下标是从1开始的
