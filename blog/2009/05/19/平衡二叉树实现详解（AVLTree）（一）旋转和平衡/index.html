平衡二叉树实现详解（AVLTree）（一）旋转和平衡

May 19th, 2009

(hplonline)2009.5.19

完整的实现，可以参考前面，这篇将分解各个细节部分。

二是这玩意写起来确实很费事，趁还记忆良好，记着，老了也好翻一翻。

对AVL有基本了解，
但在实现时卡住的同学。

由于这个结构是很对称的，所以都只拿一半来讲。

基本记号约定：

方形：表示节点
三角形：表示树，可能为空树
h(x)：x节点的高度，即x到最远子孙的路径长度，叶节点为0，空树为-1
left,right,parent：左，右孩子，父亲，根节点父亲为NULL
b(x)：x节点的平衡因子，定义为h(x->left) - h(x->right)

AVLTree类的成员函数中，
对外开放的接口习惯用每个单词大起命名。
私有函数用下划线起小写命名。

AVL的目标：

取最大，取最小，取前驱，取后继，插入，删除，查找，遍历

总体实施方案：

以上操作为一般搜索树都支持的，
实现的时候前半截和BST一样，
核心在于随时保持对每个节点x，b(x) = -1,0,1之一。
这样就能保证对数级的深度了。

实现上，主要注意的是父子的关系。
上图仅表达出了旋转的轴心parent，
而旋转后要注意维护parent的父节点ancestor和子树的关系。
同时判断parent为整棵树根的情况。

PANode AVLTree::_rotate_right(PANode p ){
    PANode child = p->left ;
    p->left = child->right ;
    if ( child->right ) child->right->parent = p ;
    child->right = p ;
    PANode ancestor = p->parent ;
    if ( ancestor ) {
        if ( ancestor->left == p ) ancestor->left = child ;
        else ancestor->right = child ;
    }else{
        root = child ;
    }
    child->parent = ancestor ;
    p->parent = child ;
    return child ;
}

旋转操作是平衡操作的子步骤，
在该步骤中不用考虑平衡因子的调整。
把平衡因子的问题交给平衡操作和插入删除操作配合完成。

左平衡操作一。左左型：

parent是第一个失去平衡的节点。
cur是导致失去平衡的节点。
在插入操作中，可以理解为是cur的增高。

有点绕口。。

如下：

实际情况中，其实就是ABCD都为空，而cur是新插入到child的节点。
这将导致h(parent) = 2，失去平衡。

在cur改变之前，该树一定是平衡的，而cur的插入导致了该树不平衡。
此时的b(child) = 1

旋转后如下，重点就是讨论旋转后的平衡因子的变化。

由于ABCD为空，这时，显然：
B(parent) = B(child) = B(cur) = 0

对应的代码就是：
    case 1:
        re = _rotate_right(p) ;
        re->balance = 0 ;
        re->right->balance = 0 ;
        break;

当然，左左出现，还有可能是在删除过程中，由于A树的降低导致的。
那么还是这幅图：


当h(cur) - h(B) = 1的时候，同上面的讨论。
而当h(cur) - h(B) = 0的时候，
由于：
h(child) - h(A) = 2
h(cur) + 1 - h(A) = 2
h(cur) - h(A) = 1

b(child) = h(cur) - h(parent) = h(cur) - (h(B) + 1) = -1
b(parent) = h(B) - h(A) = h(cur) - h(A) = 1

对应的代码就是：
    case 0:
        re = _rotate_right(p) ;
        re->balance = -1 ;
        re->right->balance = 1 ;
        break;

左平衡操作二。左右：

既为左右，那么h(cur) - h(B) = 1
先左旋：

再右旋：

此时的情况是：
h(cur) = h(B) + 1;因为是“左右”情况，b(child) = -1
h(A) = h(child) - 2 = h(cur) - 1 = h(B)
h(C)<=h(B),h(D)<=h(B);否则child就已经不平衡了
所以b(parent) = h(child) - h(parent) = max{h(B),h(C)} - max{h(D),h(A)} = 0

而child和parent的平衡因子将根据C,D的关系调整。

比如h(C) = h(D)的时候，必有h(C) = h(B)，否则A的降低不足以使parent失衡。
于是b(child) = b(parent) = 0 ;

若由于插入失衡，那么设C增高。于是有：
h(C) = h(B) = h(D) + 1

对应代码：
    case -1:
        _rotate_left(child) ;
        re = _rotate_right(p) ;
        tmp = re->balance ;
        re->balance = 0 ;
        switch(tmp){
        case 0:
            re->left->balance = 0 ;
            re->right->balance = 0 ;
            break;
        case 1:
            re->left->balance = 0 ;
            re->right->balance = -1 ;
            break;
        case -1:
            re->left->balance = 1 ;
            re->right->balance = 0 ;
            break;
        }
        break;

完整的左平衡代码如下：

PANode AVLTree::_balance_left(PANode p ){
    PANode child = p->left ;
    PANode re ;
    int tmp ;
    switch ( child->balance ) {
    case 1:
        re = _rotate_right(p) ;
        re->balance = 0 ;
        re->right->balance = 0 ;
        break;
    case -1:
        _rotate_left(child) ;
        re = _rotate_right(p) ;
        tmp = re->balance ;
        re->balance = 0 ;
        switch(tmp){
        case 0:
            re->left->balance = 0 ;
            re->right->balance = 0 ;
            break;
        case 1:
            re->left->balance = 0 ;
            re->right->balance = -1 ;
            break;
        case -1:
            re->left->balance = 1 ;
            re->right->balance = 0 ;
            break;
        }
        break;
    case 0:
        re = _rotate_right(p) ;
        re->balance = -1 ;
        re->right->balance = 1 ;
        break;
    default:;
        //assert(“false”) ;
    }
    return re ;
}
