枚举型，结构体，名字空间，静态转换，习惯问题

Jun 29th, 2009

(hplonline)2009.6.29

（环境为VC6.0)

例一：

定义如下结构。

struct sa{
    enum {
        e1,e2,e3
    }val ;
}a ;

struct sb{
    sa a ;
}b ;

做赋值：
a.val = e1 ;

编译器会报错，说e1没有定义。

办法：
    a.val = sa.e1 ;
    a.val = a.e1 ;
    a.val = sa::e1 ;

最后一个比较好理解，就是指定名字空间，
一个结构体，一个类，都相当于一个独立的名字空间。
第二个，引用方式很像成员变量。不管他。。
语言这东西本来就没为什么的，能用就行。
第一个看起来就更别扭了。。
如果说第二个是实例化的a，把e1当作成员变量的低位，
那么第一个用法，在.前面放的是结构体名（类名）。。。
暂且记下这个用法，能用就行了。

那么对b.a.val赋值呢。其实就差不多了：
    b.a.val = sa::e1 ;
    b.a.val = b.a.e1 ;
    b.a.val = sa.e1 ;

第二个用法因为是实例化的对象名，所以要从b开始跟着走。
如果发现用b.a.val = a.e1 ;也是可以的，
是因为最开头我定义了一个叫a的这个东西。

例二：

就是一个嵌套。
反正编译过了，说明可以这样用而已。

struct sb{
    struct sa{
        enum {
            e1,e2,e3
        }val ;
    } a;
}b ;

int main(){
    b.a.val = sb::sa.e1 ;
    b.a.val = b.a.e1 ;
    b.a.val = sb::sa::e1 ;
    return 0 ;
}

例三：

前面提到过，名字空间方式引用很好理解，
但形如成员变量的引用，总觉得有点别扭。
但这种引用方式是有意义的。比如：

struct {
    struct {
        enum {
            e1,e2,e3
        }val ;
    }a ;
}b ;

int main(){   
    b.a.val = b.a.e1 ;
    return 0 ;
}

这种不先定义类型，直接用结构体定义变量的习惯在老点的代码里面有不少。
这个时候没有结构体的名字，只有通过实例化的变量去索引到相关的没举行的值了。

例四：

这下来个静态转换，上面的这种定义习惯就出现尴尬了。

在例三的代码上做一句：
b.a.val = 1 ;

会有报错：
error C2440: ‘=’ : cannot convert from ‘const int’ to ‘enum sb::sa::ee’
        Conversion to enumeration type requires an explicit cast (static_cast, C-style cast or function-style cast)

说明了，从其他类型到枚举型要用显示转换。

但是根据例三的定义，各个类型都没有名字，这下就相当尴尬了。

虽然c里面有sizeof，却没有个typeof来用。

所以，好的习惯还是把该有的东西都带上名字，少用直接定义。

struct sb{
    struct sa{
        enum ee{
            e1,e2,e3
        }val ;
    }a ;
}b ;

int main(){   
    b.a.val = static_cast<sb::sa::ee>(1) ;
    return 0 ;
}

关于static_cast，可以参考我前面关于C++类型转换的笔记
