switch..case..标签..排版

Jul 15th, 2009

(hplonline)2009.7.15

上一篇不知道怎么提到排版的问题了。
当然就说到了强大的alt+f8。
于是又想到了跟switch有关的一点猥琐事情。

例一：

下面的程序的输出是？

#include <stdio.h>
int main(){
    int a = 0 ;
    switch(a){
    case 1:puts(“aaaa”) ;break ;
    defau1t:puts(“bbbb”) ;break ;
    } ;
   return 0 ;
}

其实就是什么都不输出。。
虽然default打错了，
但是他的欺骗性在于defau1t会被认为是标签，
于是这段是符合语法的。

至少有三个办法看出这个问题。
1.放到IDE里面去，没有被标注高亮的颜色。
2.特明显的就是VC6的alt+f8。
被排过后，标签始终是顶到行首的，一眼就看出来了。
3.编译器要给出一个：
warning C4102: ‘defau1t’ : unreferenced label


例二：

不用管他实现的是啥功能，和分析其他逻辑，
这个事情可以仅仅从排版技术上解决。

比如拿到后往VC里一放，ctrl+a，alt+f8。

include<iostream>

include<iomanip>

using namespace std;
int main()
{
    enum color{red,yellow,blue,white,black};
    color pri;
    int i,j,k,n=0,loop;
    for(i=red;i<=black;i++)
        for(j=red;j<=black;j++)
            if(i!=j)
            {for(k=red;k<=black;k++)
            if((k!=i)&&(k!=j))
            {
                n=n+1;
                cout<<setw(3)<<n;
                for(loop=1;loop<=3;loop++)
                {switch(loop){
case1:pri=color(i);break;
case2:pri=color(j);break;
case3:pri=color(k);break;
        default:break;
                }
                switch(pri)
                {
                case red:cout<<setw(8)<<”red”;break;
                case yellow:cout<<setw(8)<<”yellow”;break;
                case blue:cout<<setw(8)<<”blue”;break;
                case white:cout<<setw(8)<<”white”;break;
                case black:cout<<setw(8)<<”black”;break;
                default:break;
                }
                }
                cout<<endl;}}
            cout<<”total:”<<n<<endl;

}

显然，中间标红的三行会格外的扯人眼球啊。
标签的这种性质使得我们很容易发现他。

于是可以很容易的想到，作者是想做
case 1:
case 2:
case 3:
把空格给打掉了。。。
这一掉就成了普通的标签了。
