BrainFuck（一）解释器

Jul 29th, 2009

(hplonline)2009.7.29

前天听说了brainfuck这种强大的语言，
在fuck自己brain的同时也fuck别人的brain。

从网上摘抄一段描述：

这种语言基于一个简单的机器模型，除了指令，这个机器还包括：一个以字节为单位、被初始化为零的数组、一个指向该数组的指针(初始时指向数组的第一个字节)、以及用于输入输出的两个字节流。
下面是这八种状态的描述，其中每个状态由一个字符标识：
字符 含义
＞ 指针加一
＜; 指针减一
+ 指针指向的字节的值加一
- 指针指向的字节的值减一
. 输出指针指向的单元内容(ASCII码)
, 输入内容到指针指向的单元(ASCII码)
[ 如果指针指向的单元值为零，向前跳转到对应的]指令的次一指令处
] 如果指针指向的单元值不为零，向后跳转到对应的[指令的次一指令处

规则简单，则变化更为灵活多样，就像围棋一样。

前面若干条都很好理解，
最后两条的话，用C的循环去对应，就能想通了：

[ while (*ptr) {
] }

在以上指令的基础上，为了能够进行调试，
我自己加一个’@’，用来输出当前整个数组的情况。

我的解释器：

网上有不少地方提到编译器，总觉得听上去有点别扭，
看他们所干的事情实际上就是解释的功能，
甚至连函数名都是interpret之类的。

我这个解释器的一个特点是用STL实现，
所以那个数组可以是任意伸缩的。
但是由于在语言的定义里面没有规范数组下标为负的情况，
所以遇到的时候，仅仅是简单报告错误。

第二个特点就是没有用递归实现[]。
由于[]在逻辑行为上很像while循环，
我在网上看到过的一份代码就是采用每次找到一对括号，
把这段截取成一个新的字符串，然后递归调用执行。
我是直接从上面的定义出发，让程序按照那样的规定跳转。
在实际操作上，先用map<string::iterator , string::iterator>
的结构把匹配的括号保存下来，即可实现高速查找配对括号的位置。
而这个预处理只占用一次线性的时间。

第三个也将就算个特点，就是忽略空格、tab、换行这些字符。
于是可以把程序写得更有结构一点。。

代码：

pragma warning(disable:4786)

include <iostream>

include <fstream>

include <stdio.h>

include <stack>

include <list>

include <string>

include <map>

using namespace std ;

bool work_brackets(string &code , map<string::iterator , string::iterator> &brackets){
    stack<string::iterator> st ;
    string::iterator it ;

    brackets.clear() ;
    while ( !st.empty() ) st.pop() ;
    for ( it = code.begin() ; it != code.end() ; it ++ ){
        if ( it == ‘[’ ){
            st.push(it) ;   
        }
        if ( it == ‘]’ ){
            if ( st.empty() ) return false ;//brackets unbalanced
            brackets[it] = st.top() ;
            brackets[st.top()] = it ;
            st.pop() ;
        }
    }

    if ( st.empty() ) return true ;
    else return false ;
}

void interpret(string &code){
    list<char>::iterator pointer ;
    list<char>::iterator tmpit ;
    list<char> array ;
    map<string::iterator , string::iterator> brackets ;

    if ( !work_brackets(code , brackets) ){
        cerr<<”brackets is unbalanced”<<endl ;
        return ;
    }

    array.clear() ;
    array.push_back(char(0)) ;
    pointer = array.begin() ;
    string::iterator it = code.begin();

    while ( it != code.end() ){
        switch(it){
        case ‘+’:
            ++ pointer ;
            break ;
        case ‘-‘:
            – pointer ;
            break ;
        case ‘<’:
            if ( pointer != array.begin() ) {
                – pointer ;
            }else{
                cerr<<”pointer down overflow”<<endl ;
            }
            break ;
        case ‘>’:
            tmpit = pointer ;
            ++ tmpit ;
            if ( tmpit == array.end() ){
                array.push_back(char(0)) ;
            }
            ++pointer ;
            break ;
        case ‘[‘:
            if ( pointer == 0 ) {
                it = brackets[it] ;
            }
            break ;
        case ‘]’:
            if ( pointer != 0 ) {
                it = brackets[it] ;
            }
            break ;
        case ‘.’:
            cout.put(pointer) ;
            break ;
        case ‘,’:
            pointer = cin.get() ;
            break ;
        case ‘@’:
            cout<<endl<<”———–memory:”<<endl ;
            for ( tmpit = array.begin() ; tmpit != array.end() ; tmpit ++ ){
                if ( tmpit == pointer ){
                    cout<<’(‘<<int(tmpit)<<”) ” ;
                }else{
                    cout<<int(tmpit)<<’ ’ ;
                }
            }
            cout<<endl<<”———–end of memory”<<endl ;
            break ;
        default:
            if ( strchr(”     \r\n” , it) ){
                //ignore blanks
            }else{
                cerr<<endl<<”bad charactor (“<<it<<”) in file”<<endl;
            }
            break ;
        }
        it ++ ;
    }
    cout<<endl<<”program end.”<<endl;
}


int main(int argc , char argv[]){
    string code ;
    string tmp ;
    ifstream fin(argv[1]) ;

    if ( fin.is_open() ){
        code = “” ;
        while ( !fin.eof() ){
            fin>>tmp ;
            code += tmp ;
        }
        interpret(code) ;
    }else{
        cout<<”file open error”<<endl;
    }
    return 0 ;
}

一个样例：

>+++++++@++[<++++++++>-]<.>++++
+++[<++++>-]<+.+++++++..+++.[-]>++++++++[<++++>-]<.
>+++++++++++[<+++++>-]<.>++++++++[<+++>-]<.+++.——.——–.[-]
>++++++++[<++++>-]<+.[-]++++++++++.

保存成一个文件，把文件名用第一参数传给上面的解释器就行了。
如：bf 1.txt

结果：

———–memory:
0 (7)
———–end of memory
Hello World!

program end.

除了标红的@是我自己加进去的便于调试的东西外，
这段程序就是网上很流行的那个hello world。
其中每一段调试信息输出都由——-分节。

数组是从左到右输出，带有括号的表示当前指针所指向的单元。

这下把这个小东西做好了，就可以耍一耍这种哦哦那个语言了。。
