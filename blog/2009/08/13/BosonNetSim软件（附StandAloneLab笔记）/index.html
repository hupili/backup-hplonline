Boson NetSim软件（附Stand Alone Lab笔记）

Aug 13th, 2009

(hplonline)2009.8.13

Boson NetSim：

相当好用的一个软件。
据说仿真从CCNA到CCNP的实验都足够了。




软件带有图形界面的拓扑设计，比较方便。
仿真的时候，所有的运行过程都在这个软件内，
所以失真度还是比较大，
不过仿真二字的言下之意就是有失真。

值得一提的是这个lab navigator，
按照难度分成了若干类。

之前拿这个软件的时候，
同学给的包里顺带捎了一份国人做的入门教程。
拿到当天也就试了下，
不过除了照这一阵乱敲之外，
硬是没有懂起啥东西。

而这个软件自带的lab view相当不错，
有不长不短的解释，
命令也很详细，不像国人的资料，
老是有很多缩写，
还得自己输进去来看全称。

把里面的Stand Alone Labs做完，
就大致知道了交换机和路由的基本玩法，
然后可以适时进阶，或者找点更专业，
更逼真的仿真软件来玩玩。
或者这个时候，再看看一些市面流行的实验，
基本也能看懂。

这几天把入门级的lab都做完了，
发现以前用的tp-link之类的路由器纯属家用型玩具。
cisco的货果然好玩，内容非常丰富。
要是有实物就更好玩了。

Stand Alone Lab笔记：

（仅仅是为了玩玩设备，并非为考试准备，
所以是在没有相关理论基础的情况下做的，
有的地方可能看起来很可笑）

1.
配置好端口后，可以先ping一下自身该端口的地址，
确定该端口是否通畅。
或者show interfaces xxx yyy

2.ppp+chap

enable password
username ??? password ???
ppp authentication chap
encapsulation ppp

当出现A端可以ping通自身，而B端连自身都无法ping通的时候。
在B端show int se0可以看到：
Serial0 is up, line protocol is down
表示se0已经被打开，但是没有得到对方的认证。

这个时候试着在A端重设密码：
enable password ???

或者在B端重设用户名+密码：
username xxx password yyy

3.reload命令
感觉貌似有问题，
在配置过frame-relay的dlci之后，
reload并不清除。
而在之后的地方又没有找到删除该号的方法。
可能是仿真系统的问题。

4.
show ip int e0
可以看到access-list有关的信息。
形如：
Outgoing access list is not set
Inbound access list is not set

show int e0
看不到

5.
wildcar mask
通配符的掩码
为1的部分表示用来支持通配的位。
比如：
permit tcp 24.17.2.16 0.0.0.15 any eq telnet log
即是把24.17.2.16子网来的telnet包都放行。
这个通配符可以视作是子网掩码取反
如果通配符为0.0.0.0，即是要完全匹配地址，
他和host关键字是一个意思。

6.访问控制表
在conf t状态下：
access-list创建的是无名字的表
ip access-list extended xxxxx创建的是有名字的表

7.
access list 的最后有个隐含的阻止一切数据包的条目

8.
switch 的 interfaces 命令后面，
可以跟vlan配置关于vlan的ip地址，
也可以跟range指定一个范围：
int range fa0/2-5

但是这个模拟系统中？没有提示。
不知道是设备本身的问题，还是模拟系统的问题。

9.
(switch)
在#提示下，可以vlan database进入vlan的配置。
在#(config)提示下，直接vlan也可以

10.
貌似不同型号的交换机在端口配置的地方区别比较大。
高端型号有switchport项，里面可以配置若干内容。
