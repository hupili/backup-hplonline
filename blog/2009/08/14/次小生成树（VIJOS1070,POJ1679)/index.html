次小生成树（VIJOS1070,POJ1679)

Aug 14th, 2009

(hplonline)2009.8.14

结论：
替换最小生成树中的任何一条边，将使总代价变大

根据路径最优条件，（参考这里）
MST上的任两点(i,j)的树上路径的所有边的权重，
均小于w[i][j]。所以结论显然。

次小生成树1，简单解法：

有个比较直观的方法。
依次把MST上的边删掉，然后再做MST。
这样的做就是编码简洁，有模块可以重用。
效率O(n*O(MST))

次小生成树2，优化解法：

既然每替换出去一条边，都将使代价变大。
所以求次小生成树，只需要替换出去一条边即可。

枚举不在树上的边eij，
树上路径上的边权都不大于w[i][j]。
选取其中最大的与eij替换，
得到新树的权重。
选取替换后最小的一个就是次小生成树了。
这个过程可以做到O(m)

然后就是计算出path_max[i][j]表示i，j树上路径中的最大边权。
在计算MST的时候，点是依次加入到树中。
当前加入的是j，与他相连的是i。
那么，对所有在树上集合中的k：
path_max[k][j] = max{ path_max[k][i] , w[i][j] }
因为path_max[][]的每个元素都要计算，且仅一次。
累加起来的复杂度是O(n2)

 总的复杂度：
O(MST)+O(n2)+O(m)

 MST的计算采用任何一种方法都可以。

VIJOS1070：

题目就是很明摆着的次小生成树了。
关于无解的输出是：
1.首先如果MST都无解，那么显然也不存在次小
（判断依据为：有没有成功加进n-1条边）
2.MST有解的情况下，如果m == n-1，
那么不存在次小生成树，否则一定存在。
（至少都可以进行一次替换）

关于这道题的WA。
首先是把：
ans_mst - path_max[i][j] + map[i][j]
写成了：
ans_mst + path_max[i][j] - map[i][j]

。。。

然后是求次小的时候，判断边：
if ( !edge_in[i][j] && map[i][j] != MAXINT ){

最初写的是

if ( edge_in[i][j]) 。。。。

注意是枚举不在树中的边，然后要记得判断是不是这个边不存在。
由于我这里是用MAXINT来表达的边不存在，
当不存在的时候，下面的加法中会溢出成负数，反而更小。。

VIJOS1070代码：

include <stdio.h>

const int MAXN = 510 ;
const int MAXM = MAXN * MAXN ;
const int MAXINT = 2147483647 ;

int path_max[MAXN][MAXN] ; //树上路径的最大值
int d[MAXN] , pre[MAXN] ; //最短边长，前驱点
int in[MAXN] , num_in ;
bool isin[MAXN] ;//顶点是否在树内
bool edge_in[MAXN][MAXN] ;//边是否在树内

int map[MAXN][MAXN] ;
int n , m ;
int ans_mst ;

int getmin(int a , int b){
return a < b ? a : b ;
}

int getmax(int a , int b){
return a > b ? a : b ;
}

//最小
int mst1(){
int i , j ;
int min , min_j ;
int ans = 0 ;
num_in = 1 ;
in[0] = 1 ;
isin[1] = true ;
d[1] = 0 ;
for ( i = 2 ; i <= n ; i ++ ){
d[i] = map[1][i] ;
pre[i] = 1 ;
}
for ( i = 1 ; i < n ; i ++ ){
min = MAXINT ;
for ( j = 1 ; j <= n ; j ++ ){
if ( !isin[j] && d[j] < min ) {
min = d[j] ;
min_j = j ;
}
}
if ( min == MAXINT ) return -1 ;//无法形成树

//更新树上路径表
for ( j = 0 ; j < num_in ; j ++ ){
if ( in[j] != pre[min_j] )
path_max[min_j][in[j]] = getmax(path_max[pre[min_j]][in[j]] , map[min_j][pre[min_j]] );
else
path_max[min_j][in[j]] = map[min_j][pre[min_j]] ;
path_max[in[j]][min_j] = path_max[min_j][in[j]] ;
}

in[num_in] = min_j ;
num_in ++ ;
ans += min ;
isin[min_j] = true ;
edge_in[min_j][pre[min_j]] = true ;
edge_in[pre[min_j]][min_j] = true ;

//更新最短边记录
for ( j = 1 ; j <= n ; j ++ ){
if ( !isin[j] && map[min_j][j] < d[j] ) {
d[j] = map[min_j][j] ;
pre[j] = min_j ;
}
}

}
ans_mst = ans ;
return ans ;
}

//次小
int mst2(){
int ans = MAXINT ;
int i , j ;
if ( m == n - 1 ) return -1 ;
for ( i = 1 ; i <= n - 1 ; i ++ ){
for ( j = i + 1 ; j <= n ; j ++ ){
if ( !edge_in[i][j] && map[i][j] != MAXINT ){
ans = getmin(ans_mst - path_max[i][j] + map[i][j] , ans) ;
}
}
}
return ans ;
}

int main(){

int i , j ;
int x , y , c ;
scanf(“%d%d” , &n , &m) ;
for ( i = 1 ; i <= n ; i ++ ){
for ( j = 1 ; j <= n ; j ++ ){
map[i][j] = MAXINT ;
}
}
for ( i = 0 ; i < m ; i ++ ){
scanf(“%d%d%d” , &x , &y , &c) ;
map[x][y] = c ;
map[y][x] = c ;
}
int ret = mst1() ;
printf(“Cost: %d\n” , ret) ;
if ( ret == -1 ) printf(“-1\n”) ;
else printf(“Cost: %d\n” , mst2()) ;

return 0 ;
}

POJ1679：

问的是MST是否唯一。
那么求出最小生成树和次小生成树，
如果权重相等，当然就是不唯一的。

只不过这个想法太直观了，
干了不少多余的事情。
直接在MST的计算过程中，就可以判断出来了。

拿PRIM来说：

每次添加点的时候，要找出关联边最小的。
如果这个最小值不唯一，那么就可以直接指出MST不唯一了。

拿KRUSCAL来说：

由于边已经按权值排序了，
那么添加一个边的时候，
把和他权值相同的边都试添加一次，
如果有多余一条边可以添加进，也指出不唯一。
