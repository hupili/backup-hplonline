预测手段，定参手段

Sep 2nd, 2009

(hplonline)2009.9.1

据说霍金这样说过：
我注意过，即便是那些声称一切都是命中注定的而且我们无力改变的人，在过马路之前都会左右看。

所以有的时候，虽然我们很明确是在装13，
但毕竟得拿出方案，装得彻头彻尾。

先看一组数据：
25.80    42.60    24.70    3.90    3.00    0.00   
15.30    20.20    49.80    9.70    1.90    3.10   
12.20    24.90    43.60    13.30    2.60    3.40   
11.50    24.10    52.80    8.30    1.70    1.60   
5.20    39.80    35.20    9.50    6.20    4.10   
5.60    32.80    35.60    16.60    4.40    5.30   
5.90    33.10    34.70    14.00    5.50    6.80   
4.40    44.00    28.30    10.00    3.20    10.00   
4.70    41.50    31.30    6.40    5.80    10.30   
1.20    26.90    39.90    14.80    5.90    11.30   

横着的每一行加和为100，其实代表的是各类脑残的百分比。
而竖着看表示各年的情况。

从左往右，脑残程度依次增加。
比如第一行的25.80，代表一般思维正常的人；
42.60属于2级脑残，各种蛋疼围观群众属于这类；
而3、4、5等等是级别越高越严重；
最后一列是劣5类脑残，属于无可救药类，
hplonline正好属于这一列。

拿到这些数据的目的并不是放在那乘凉的，
显然是要来做做预测之类。
比如，我们可以问问劣5类脑残所占的比例发展趋势如何？
也许从数值上可以让我看到希望。

matlab的fminsearch可谓强大，
配合各种准则可以拟合出不同需求的东西来，
这是一个通用的方法。

一。分析脑残比例的变化方式

一般来说，经过整治，重度脑残是可以变轻的。
又一般来说，经过洗脑，轻度脑残也是可以变重的。
于是可以给出参数aij表示第i类脑残化为第j类脑残的比例，
很显然的，aij对j求和应该是为1的，
因为地球人都属于某种脑残，不可能凭空变成火星人。

另外一点，脑残的变化应该是有过渡性的，
所以不考虑突然加重两级这样的不合理情况。
从理论上说，跳级只有两种可能：
1.被hplonline洗脑后可能直接到达劣5类
2.春歌出手拯救，自然可以从各个级别直接降为1类。

但是，被hplonline洗脑是需要一个长期过程的，
既然是进行短期预测，自然不用考虑这种情况。
而春歌要保持元气去拯救地球，一般不轻易出手，
那么，这种情况也不考虑了。

由于只是进行方法展示，转移函数从简，就用个线性的。
记列向量y(n)为各级别脑残所占比例。
如果给出一个转移矩阵：

A = [
    a11 a21   0   0   0   0;
    a12 a22 a32   0   0   0;
    0   a23 a33 a43   0   0;
    0   0   a34 a44 a54 0;
    0   0   0   a45 a55 a65;
    0   0   0     0 a56 a66;
] ;

那么，y(n)=A*y(n-1)

等于说把这个A矩阵给确定了，
就可以YY出未来脑残的发展趋势。

二。通用的定参手段

fminsearch来干这勾当其实是很合适的，
只要转换一些观念上的东西，就很好套用。

最基本的就一句话：
[x fval flag] = fminsearch(‘myfunc’ , x0)

这里的x0是一个向量，依次排上了待定的各个参数
即，x0=[a12，a23，a34，a45，a56，a21，a32，a43，a54，a65]T

而myfunc是一个需要自己构造的函数，
返回一个值，告诉fminsearch这组参数效果怎样。

这里的myfunc可以这样写：
function err = myfunc(x)
1.从x向量中取得各个参数，构造出转移矩阵
2.递推出预测序列y(n)=A^n*y(0)
3.根据原始序列p(n)和预测序列y(n)，给出评价标准

这种方法实际上可以用于蹂躏任何数据，
首先，给出从之前序列推得之后值点的方式，
然后，给出拟合好坏的标准，
最后，fminsearch就把参数搞出来了。。。

三。各种评价标准

一般这类数据拟合喜欢用最小二乘准则，
原因是该准则比起残差绝对值和最小准则，
以及chebyshev准则来说，容易求导，也就容易推出理论的计算公式。

而我们现在要用的方法是一种搜索，
选用以上各种准则，甚至自己发明的准则来说，
都是一样的，仅仅是计算评价标准时候耗时有点差异。

可以先来观察我试过的几个标准。
首先用p表示原序列：
p = [
25.80    42.60    24.70    3.90    3.00    0.00   
15.30    20.20    49.80    9.70    1.90    3.10   
12.20    24.90    43.60    13.30    2.60    3.40   
11.50    24.10    52.80    8.30    1.70    1.60   
5.20    39.80    35.20    9.50    6.20    4.10   
5.60    32.80    35.60    16.60    4.40    5.30   
5.90    33.10    34.70    14.00    5.50    6.80   
4.40    44.00    28.30    10.00    3.20    10.00   
4.70    41.50    31.30    6.40    5.80    10.30   
1.20    26.90    39.90    14.80    5.90    11.30   
]’;
注意最后有一个转置。

y和p是同样维度的，只是在对应位置上都变为预测数据。

如果选取残差平方和最小，就是这样：
err = sum(sum((p - y).^2)) ;



这个时候，考虑到有的脑残比例本来就很小，
而另外的脑残比例占得很大，
仅仅是用残差平方和的话，
会更偏重于大者。
所以，可以考虑把准则换为相对残差和最小：
err = sum(sum(((p - y)./p).^2)) ;



这样出来的图还有个问题，
就是从数据来看，
本来1类脑残应该是势力越来越单薄才对，
而图上的曲线已经趋于平稳。
所以考虑这样一个时间依赖性的准则，
即我们更倾向于相信最近的数据。
于是对这10年的残差进行加权，
让越近的残差的重量越大。
这样出来的曲线会更“将就”最近的数据。
出于简单考虑，直接按照1-9线性加权就是了：
err = sum(sum(((p - y)./p.*(ones(6,1)*[1:9])).^2)) ;


四。初始值的问题

fminsearch在搜索的时候，会陷入局部极值点，
所以一般的做法是先随机出一系列的参数，
选取其中效果最好的点，再开始搜索。

不过既然是随机，所以每次运行的时候都可能出现不同的结果，
因为随到的初始点附近有可能有局部极小值。
但从fminsearch的返回fval可以得到自己定的评价标准，
如果两次运行的结果不同，显然选其中fval较小的。

主过程就长成这样：

N = 10000 ;
flag = 0 ;
while flag == 0
    bf = 1e6 ;
    for i = 1:N
       x = rand(1,10) ;
       f = myfunc(x) ;
       if f < bf
         bf = f ;
         bx = x ;
      end
    end
    [x fval flag] = fminsearch(‘myfunc’ , bx) ;
end
x,fval

五。myfunc的实现细节

既然我前面是随机出来的参数，
那么很有可能有某列加起来已经大于1的情况。
这个时候，显然是不满足要求的，就强制返回一个极大值，
这个值形象地说起来就像一堵墙，
fminsearch就会向着离墙越远的地方走。
（这同时说明了脱离墙不仅是人们美好的愿望，
一个matlab的小小函数也是如此）

如果发现加和满足要求，那么就计算出同一列的另外的参数。
比如：
if a21 + a23 > 1
    err = MAX_ERR ;
    return ;
else
    a22 = 1 - a21 - a23 ;
end

之后把这些参数带如A矩阵，
计算出y(n)，和p(n)一起计算出评价指标返回就行。

整个函数写出来是这样：

function err = myfunc(x)
MAX_ERR = 1e5 ;
if sum(x < 0 | x > 1) > 0
    err = MAX_ERR ;
    return ;
end
a12 = x(1) ;
a23 = x(2) ;
a34 = x(3) ;
a45 = x(4) ;
a56 = x(5) ;
a21 = x(6) ;
a32 = x(7) ;
a43 = x(8) ;
a54 = x(9) ;
a65 = x(10) ;
a11 = 1 - a12 ;
a66 = 1 - a65 ;
if a21 + a23 > 1
    err = MAX_ERR ;
    return ;
else
    a22 = 1 - a21 - a23 ;
end
if a32 + a34 > 1
    err = MAX_ERR ;
    return ;
else
    a33 = 1 - a32 - a34 ;
end
if a43 + a45 > 1
    err = MAX_ERR ;
    return ;
else
    a44 = 1 - a43 - a45 ;
end
if a54 + a56 > 1
    err = MAX_ERR ;   
    return
else
    a55 = 1 - a54 - a56 ;
end
A = [
    a11 a21   0   0   0   0;
    a12 a22 a32   0   0   0;
    0   a23 a33 a43   0   0;
    0   0   a34 a44 a54 0;
    0   0   0   a45 a55 a65;
    0   0   0     0 a56 a66;
] ;
p = [
25.80    42.60    24.70    3.90    3.00    0.00   
15.30    20.20    49.80    9.70    1.90    3.10   
12.20    24.90    43.60    13.30    2.60    3.40   
11.50    24.10    52.80    8.30    1.70    1.60   
5.20    39.80    35.20    9.50    6.20    4.10   
5.60    32.80    35.60    16.60    4.40    5.30   
5.90    33.10    34.70    14.00    5.50    6.80   
4.40    44.00    28.30    10.00    3.20    10.00   
4.70    41.50    31.30    6.40    5.80    10.30   
1.20    26.90    39.90    14.80    5.90    11.30   
]’;
p0 = p(:,1) ;
p = p(: , 2:end) ;
y = [] ;
for i = 2:10
    p0 = A * p0 ;
    y = [y p0] ;
end
%err = sum(sum((p - y).^2)) ;
%err = sum(sum(((p - y)./p).^2)) ;
err = sum(sum(((p - y)./p.*(ones(6,1)*[1:9])).^2)) ;

这样把参数搞定后，做做矩阵乘法，画个图就是很简单的事了。

六。评价

这里只是讨论手段问题，不讲究任何理论依据，
因为这是没有依据的。
即使这个矩阵乘法看起来很像leslie模型，
甚至还可以说是改进的leslie模型，
我还是要说是没有依据的。
