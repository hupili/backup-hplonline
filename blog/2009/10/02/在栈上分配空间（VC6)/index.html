在栈上分配空间（VC6)

Oct 2nd, 2009

(hplonline)2009.10.2

有的时候，在某个函数里面要动态分配一段内存，
这段内存仅仅是作为这个函数处理所用的临时空间，
并不用在该函数之外的其他地方。

用new比malloc慢就不说了，
如果还想比malloc快一点，可以直接在栈上分配。

而栈上一般都是编译器放的局部变量。
因为在编译期间，可以计算得到该函数内部需要的空间，
进入该函数时，除了在栈上保护相关的寄存器外，
就是从esp当前的位置向下拉出一段距离，
供给局部变量使用，
并且局部变量的偏移位置是预先可以知道的。

由于C中没有直接提供对栈的操作，
只有使用内联汇编。
又由于不同环境的汇编不太一样，
加之不同编译器的栈使用策略可能也不一样。
以下的内容就选在环境VISTA.VC6.DEBUG下了。

定义了几个宏，好写起来方便一点。
顺便比较一下和用malloc分配的耗时。

程序：

#include <stdio.h>
#include <stdlib.h>
#include <time.h>

const int N = 1024 ;
const int M = 100000 ;

//在需要内存分配的地方调用
//char *p ;
//StackAlloc(512) ;
//StackPtr(p) ;
#define StackAlloc(x) __asm sub esp , x ;
#define StackPtr(p) __asm mov p , esp ;

//在函数返回之前归还内存
//否则debug模式下会报错
#define StackFree(x) __asm add esp , x ;

void fun1(int n){
    char *p ;
    StackAlloc(n) ;
    StackPtr(p) ;
    //do something ;
    StackFree(n) ;
}

void fun2(int n){
    char *p ;
    p = (char*)malloc(n) ;
    //do something ;
    free(p) ;
}

int main(){
    int i ;
    int tmp ;

    __asm mov tmp , esp ;
    printf(“%d\n” , tmp) ;
    clock_t tt = 0 ;
    for ( i = 0 ; i < M ; i ++ ){
        fun2(N) ;
    }
    __asm mov tmp , esp ;
    printf(“%d\n” , tmp) ;
    printf(“%d\n” , clock() - tt) ;

    return 0 ;
}

fun1:
1244912
1244912
3

fun2:
1244912
1244912
85

先打印出来的esp指针，
是为了说明堆栈是平衡的。。
