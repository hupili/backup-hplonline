1.命名方式

使用
#include “f2c.h”
其中包含一些变量类型和函数类型的定义

在fortran里面写作
call func(..)
在C里面写作
func_()

2.参数传递方式

要求都以“引用”传递，
在C里面表现为全部是指针。

3.字符（串）参数

在除了测试和定时代码(timing code)的所有地方，
函数关注的都仅仅是第一个字符。

引用readme里面的例子，

fortran：
#    call dpotrf( ‘Upper’, n, a, lda, info )

C：
#    char s = ‘U’;
#    dpotrf_(&s, &n, a, &lda, &info);

4.矩阵的主序问题

在fortran里面是列主序，
C中是行主序。

C中调用的正确方法应该是，
申请一个一维数组，
然后按照列主序给元素赋值。

（原readme指出，
a[M][N]这样的二维定义，
可能被编译器实现为不连续的空间。
即Iliffle向量（ECP中的叫法）的方式。）


手册：
http://www.netlib.org/lapack/lug/

使用dgeev_的问题：
https://icl.cs.utk.edu/lapack-forum/viewtopic.php?f=2&t=661

手册中关于参数的解释：
http://www.netlib.org/lapack/lug/node110.html

———-参数传递的习惯

N 问题的尺寸，也即列的数量
LD?? (Leading Dimension)，即行的数量
info 0表示成功，-i表示第i个参数错误，+i计算错误
work 工作空间
lwork 工作空间的大小，可以传入-1以从work返回需要的空间
(这个功能一直没有尝试成功喃。。。）

》》环境搭建和第一个简单程序（VC6)

我就直接用的下面这个编译好的库：
http://www.netlib.org/clapack/CLAw32.zip
在对应的地方去把clapack.lib和blas.lib复制到工程目录下。

把clapack.h复制到工程目录下（从http://www.netlib.org/clapack/下载）


#include < stdio.h>

#pragma comment(lib , “blas.lib”)
#pragma comment(lib , “clapack.lib”)

#include “f2c.h”
#include “clapack.h”

int main(void)
{
    /* 3x3 matrix A
     * 76 25 11
     * 27 89 51
     * 18 60 32
     */
    double A[9] = {76, 27, 18, 25, 89, 60, 11, 51, 32};
    double b[3] = {10, 7, 43};

    int N = 3;
    int nrhs = 1;
    int lda = 3;
    int ipiv[3];
    int ldb = 3;
    int info;
   
    dgesv_(&N, &nrhs, A, &lda, ipiv, b, &ldb, &info);

    if(info == 0) /* succeed */
printf(“The solution is %lf %lf %lf\n”, b[0], b[1], b[2]);
    else
fprintf(stderr, “dgesv_ fails %d\n”, info);

    return info;
}
