FFT相关（三）（MATLAB）（任意基DIT递归实现)

Dec 28th, 2009

(hplonline)2009.12.28

其实有了前面基2，基4的原理。。。
任意基的FFT也就是那个样子。

详细推导和程序文件下载。

简要说明：

任意基就是把长度作因子分解：
N=a*b
于是可以通过计算a个长度为b的子序列，
然后再把结果整合得到N点的DFT。
（话说虽然DSP的期末貌似不考FFT，
但是这个任意基的原理在某课后题出现过，
只是没有点名可以用来做FFT而已）

这个过程可以递归下去，
直到b=1，就可以直接算得X=x。

详细的推导还是见下载的包吧。

另外，作为原理模拟，
像分解因数等部分写得就很土鳖了，
反正意思是到了。

关键程序段：

function X = ditany(x)
    N = length(x) ;
    if N == 1
        X = x ;
    else
        %factorize N = a * b
        flag = 0 ;
        for a = 2:fix(sqrt(N))
            if fix(N / a) == N / a
               flag = 1 ;
               break ;
            end
        end
        if ~flag
           a = N ;
        end
        b = N / a ;
        %start caculation of length b dft
        %and assemble the a sequences
        W = exp(-1j * 2 * pi / N * (0:a-1)’*(0:b-1)) ;
        %preallocation of space will be needed here
        %to accelerate the programme
        XX = [] ;
        for ii = 1:a
           XX = [XX ; ditany(x(ii:a:N))] ;
        end
        X = [] ;
        XX = XX .* W ;
        for jj = 1:a
            tmp = exp(-j * 2 * pi / N * (0:a-1) * b * (jj - 1)) * XX ;
            X = [X , tmp] ;
        end
    end
return ;

一些比较:

其实我们比较关心的是这个任意基的算法究竟能改进多少时间。
基准采用的是矩阵定义的DFT变换：
X1 = x * exp(-j * 2 * pi / N * (0:N-1)’*(0:N-1)) ;

以点数为横坐标，
加速比为纵坐标。
加速比的定义为DIT_any的耗时比上矩阵DFT的耗时。
画出散点图，并标注基准线y=1来观察。

首先来看从1到200点的散点图。


再看以2的方冪为点数的散点图。



在观察各种自然数的时候，
发现这个任意基的算法大部分时候反而更戳。

专门把2的方冪提取出来观察，
发现效率有所改进，
但似乎并不是很明显的样子。

要仔细观察其中的差异，
还得用C来实现一下。
因为MATLAB隐藏了太多的细节，
让我们无法把握每一个消耗时间的地方。
不过作为粗糙的原理模拟，
以及简易对比来说是可以的。

出现这种现象也比较好理解，
因为任意基的算法的精华在于分解。
把大的序列分解成若干小的序列，分别计算。
随便选一个数，如果是质数，
则无法发挥任意基的能力，
甚至由于其他操作引入更多的时间。
即使不是质数，
如果其中有较大的质数因子也会导致耗时较多。

由于是任意基，exp(2 * pi / N * k)的计算是逃不掉的。
不像基2基4基8，出来的是如1,j,1+j这类比较好看的数字。
