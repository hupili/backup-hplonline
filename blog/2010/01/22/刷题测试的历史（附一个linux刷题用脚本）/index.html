刷题测试的历史（附一个linux刷题用脚本）

Jan 22nd, 2010

(hplonline)2010.1.22

从最初在OJ上刷题开始，
我就在关注怎么提高测试效率。
现在回想起一些以前的行为，
感觉啼笑皆非。

1.win+TP时代

当时看到过有人写的一个bat，
比较自动化，依次测试10组数据，
利用了重定向。
使用time命令获得程序程序时间。

而windows下的这个time很戳，
需要再手动输入一个更新的时间，
不输入代表不更新。
于是需要发一个回车键给他。
一般是在一个文件里面写个 char(13)，
然后使用time <enter这样的形式解决。

2.win+FP时代

这时候比较傻，除了写点简单题，
对系统编程，对FP本身的一些支持完全不了解。

于是我们会在程序开始assign一团文件，
下面也是writeln(fout,…)。
等到该交到OJ上的时候，手动把所有fout去掉。。

后来想了下也太笨了点，
至少可以用记事本来个替换也好嘛。

3.win+TC时代

这个时候，沿用了以前FP的方法。
直到有一天，看到有人直接在控制台下粘贴，
才知道原来还可以这么玩。。

发现一般的小数据，粘贴起来很方便。
对于1写1编1A的情况来说，比较省时。

4.win+VC时代

这个时候，知道了C里面一个好用的东西，
就是freopen。。

不过由于不少时候会搞忘注释掉，
结果会很悲剧。
有的OJ报个RE之类还看得出来，
有的OJ直接报个WA，
结果在自己电脑上看，怎么都像对的。

5.后VC时代1

发现POJ上原来有个ONLINE_JUDGE宏（文章末尾）。
配合条件编译，似乎是个很不错的东西。

但是会出现一种囧况，
就是手抖把ONLINE_JUDGE打错了，
反正在本机上是未定义的，于是运行正常，
但是拿到OJ上，一样未定义。
于是那句freopen就成功把程序搞死了 。

6.后VC时代2

采取与ONLINE_JUDGE相反的思路，
定义LOCAL宏。

VC的添加位置为：
project->settings->C/C++->options
在里面写上：
/D “LOCAL”
就可以了。

这个宏用起来要安全一点，
不过编译选项是在工程中指定的，
这意味着，要保存成果的话，可能需要保存整个工程。

而我们刷题往往就一个c文件，
屁股后面还带一堆杂项就显得臃肿了。

7.linux时代

除了前面I/O问题之外，还有时间问题。
如果在程序里面用clock之类的获得时间，
交题之前也要记得注释。。
如果在程序外面 ，由系统命令来搞的话，
会比较繁琐一点，见【1】。

后来发现，在linux下，
很容易就扩展出自己需求的这些功能。

比如我做一个脚本。
编译方式选择（要不要调试信息），
重定向选择（各种I/O组合，选择是否与标准输出对比），
运行方式选择（直接运行，或者gdb）。
这些都列一个菜单出来，
选一下就行了。

另外使用linux的time来测时间很方便，
不会受到后面程序重定向的影响。

于是我现在可以这样操作。
拿到一个题，把输入放到in.txt里面
把标准输出放到std.txt里面。
然后：
vi hello.c
….
:w
:!run hello
3
如果数据一致的话，就出现：
there is no difference, congratulations!!

然后就可以拿去交了。。
不过一样可能WA，
那么我再选5方式运行，
进入gdb，跟一下。。

程序：/usr/bin/run

!/bin/bash

created by hpl

2010.1.22

to make , run , debug a program automatically

echo “select your mode”
echo “1.make,run,with redirected Input”
echo “2.make,run,with redirected Input/Output”
echo “3.make,run,with redirected I/O and compare”
echo “4.make,run,with console I/O”
echo “5.make,debug,with redirected Input”
echo “6.make,debug,with consol Input/Output”
echo “when files are required”
echo “‘in.txt’ is for input”
echo “‘out.txt’ is for output”
echo “‘std.txt’ is for standard output”

read mode
mode=${mode:-“4”}

case $mode in
“1”)
gcc -o $1 $1.c
time ./$1 <in.txt
;;

“2”)
gcc -o $1 $1.c
time ./$1 <in.txt >out.txt
;;

“3”)
gcc -o $1 $1.c
time ./$1 <in.txt >out.txt
re=diff out.txt std.txt
re=${re:-“there is no difference, congratulations!!”}
echo $re
;;
“4”)
gcc -o $1 $1.c
time ./$1
;;
“5”)
gcc -g -o $1 $1.c
gdb –args $1 “<in.txt”
;;
“6”)
gcc -g -o $1 $1.c
gdb $1
;;
*)
echo “type mode between 1 and 6 ,please!!”
esac
