【PacketTracer】NAT配置实验

Mar 18th, 2010

(hplonline)2010.3.18

用PT来配了下NAT。
关键的几个问题为：

1.测试NAT起作用

网络连好之后，选一台路由开放virtual terminal，
其他主机telnet进来，可以用show users查看用户地址。

根据用户地址就知道是NAT之前还是之后的了。

2.模拟Internet环境

这里的模拟要达到的效果是，
路由器对“Internet”方向的端口可以正常收发包，
但带有路由器对“内网”方向地址的包无法通过。

可以配置ACL来模拟。

拓扑


子网0：功能性子网，放台服务器和开了VT的路由，PC是用来测试保证本地正常访问的。
子网1：假设的Internet环境
子网2：用来实验静态NAT的子网
子网3：用来实验动态NAT的子网

实验配置过程


1.各设备连接好，按设计分配IP

2.启动路由协议

route rip
net xxxxx
net xxxxx

3.测试连通

各设备之间都能ping通。
由于PT模拟的问题，
一般每截链路的第一个包会失败。

4.配置R0的virtual terminal

line vty 0 15
password 123
login

5.测试vt的正常

各设备
telnet 192.168.0.4

R0
show users
可以看到登陆的用户地址。

6.用ACL来把子网1模拟成internet环境

R1：
access-list 1 permit 192.168.1.0 0.0.0.255
access-list 1 deny any

int f0/1
ip acc 1 in

只允许来自子网1的IP包通过。

7.从子网2和子网3的主机测试连通性。

对于子网0不可达

8.静态NAT配置

配置映射表
ip nat inside source static 192.168.2.2 192.168.1.102

配置内外接口
int f0/0
ip nat out
exit
int f1/0
ip nat in
exit

9.测试静态NAT

PC1能够连上R0，
在R0上show users可以看到
192.168.1.102连接
（地址已被转换）

10.动态NAT配置（multi-to-1 NAT)

access-list 1 permit 192.168.3.0 0.0.0.255
ip nat inside source list 1 interface fastEthernet 0/0 overload

int f0/0
ip nat out
exit
int f1/0
ip nat in
exit

11.动态NAT测试

用PC2和PC3同时连R0，成功。

从R0上show users，可以看到两个192.168.1.3（R3地址）
