【PacketTracer】dhcp中继

Mar 24th, 2010

(hplonline)2010.3.24

系列链接

dhcp中继适用的是dhcp服务器和客户机不在一个IP子网情况。
因为是在第三层做中继，所以是可以跨越多个IP子网的。

在cisco路由上，进入接口模式，可以配置一个 ip help-address。
打问号后面也说明了，是 转发UDP广播报文的目的。
(DHCP的广播都是封在UDP里面的）

练习点：

dhcp
dhcp中继
子接口（见“单臂路由”）

拓扑：


三层交换配置dhcp的地址池，
试验由路由和三层交换做dhcp中继。

三层交换机的f0/2为三层口，
下接一个子网（.3)。

路由下接两个vlan，分属两个子网。

实验步骤：

1.开启各接口，配置三层交换机和路由的IP。

MS0:
f0/1:172.16.0.1/30
f0/2:172.16.0.5/30

MS1:
f0/1:172.16.0.6/30
f0/2:172.16.3.1/24

R0:
f0/0:172.16.0.2/30
f0/1.1:172.16.1.1/24 (vlan2)
f0/1.2:172.16.2.1/24 (vlan3)

2.在MS0上配置dhcp地址池，

与普通dhcp的配置一样。

3.

交换机S0上划分vlan，
应用到接口，并且设置到R0的trunk。

4.添加各路由设备的路由。

其中R0和MS1只需一条默认路由即可，
MS0可以手动添加三条路由。

5.在R0上配置dhcp中继

int f0/1.1
ip helper-address 172.16.0.1
exit

int f0/1.2
ip helper-address 172.16.0.1
exit

6.在MS1上配置dhcp中继

与路由类似

7.检验

各主机renew一下ip，相互ping下。

——

实际上MS1有更方便的解决方案，
就是把相关端口都用在2层。
MS0-MS1的链路设为一个trunk。
类似“三层交换dhcp”。
