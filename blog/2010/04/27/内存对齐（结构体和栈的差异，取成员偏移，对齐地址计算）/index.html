内存对齐（结构体和栈的差异，取成员偏移，对齐地址计算）

Apr 27th, 2010

(hplonline)2010.4.27

印象中曾经记录过，不过居然找不到了。
联系最近听课和给别人讲东西发现的问题，重新详记一次。

环境：vc6，.cpp文件，debug编译

一。结构体对齐基本实验

程序：

include <stdio.h>

struct s1{
char c1 ;
short s ;
char c2 ;
} ;

struct s2{
char c1 ;
int i ;
char c2 ;
} ;

pragma pack(1)

struct s3{
char c1 ;
short s ;
char c2 ;
} ;

pragma pack()

pragma pack(8)

struct s4{
char c1 ;
short s ;
char c2 ;
} ;

pragma pack()

struct s5{
char c1 ;
char c2 ;
short s ;
} ;

int main(){
printf(“%d,%d,%d,%d,%d\n” ,
sizeof(struct s1) ,
sizeof(struct s2) ,
sizeof(struct s3) ,
sizeof(struct s4) ,
sizeof(struct s5)) ;

return 0 ;
}

结果：

6,12,4,6,4

分析：

s1的结果是用来说明有内存对齐这件事的。
相信很多人还记得C语言课程考试上，
问到结构体的大小时，会把它判为4。

s1和s2的对比得到一个结论：
对齐到的字节数与最大成员所占空间有关。

s3是说明可以用#pragma pack(xxx)来改变对齐到的字节数。
当然，用完了只有要用#pragma pack()来还原，
否则会对后面的结构造成影响。

s4是说，虽然用#pragma pack(8)了，但是依然按照两字节对齐。

从上面三条，可以得到的结论：
对齐到的字节数=min｛当前指定的pack值，最大成员大小｝

s5是用来和s1做对比的。
可以看到，两个结构的内容一致，仅仅是摆放顺序不一样。
这里的结论是：
x字节的成员需要对齐到x字节的偏移上。
如果这句说起有点绕口，下面一节将有更深入的探究。

二。结构体成员的偏移观察。

取得指定结构体某成员在结构体内部的偏移可以用一个宏：
#define OFFSETOF(st,mem) (size_t)(&((st*)0)->mem)

貌似在好些地方看到有人这样用了，
稍微想一下，原理也就那样，故不解释了。

程序：

include <stdio.h>

define OFFSETOF(st,mem) (size_t)(&((st*)0)->mem)

struct s1{
char c1 ;
short s ;
char c2 ;
} ;

struct s2{
char c1 ;
char c2 ;
short s ;
} ;

int main(){
printf(“%d,%d,%d\n%d,%d,%d\n” ,
OFFSETOF(s1 , c1) ,
OFFSETOF(s1 , s) ,
OFFSETOF(s1 , c2) ,
OFFSETOF(s2 , c1) ,
OFFSETOF(s2 , c2) ,
OFFSETOF(s2 , s)
) ;
return 0 ;
}

结果：

0,2,4
0,1,2

分析：

可以看到，在s1中，因为s是short型的，
需要被对齐到两字节的偏移上面。

内存对齐不仅仅考虑对齐到某个大小，
也要考虑对齐到某个偏移。

三。栈内容对齐

程序：

include <stdio.h>

int main(){
char c ;
short s ;
int i ;
float f ;
double d ;
printf(“%x,%x,%x,%x,%x\n” , &c , &s , &i , &f , &d) ;
return 0 ;
}

结果：

12ff44,12ff40,12ff3c,12ff38,12ff30

可以看出，都是对齐到4字节的。
并且前面的char和short并没有被凑在一起，
这和结构体内的处理是不同的。

在栈上，不考虑变量本身的大小，统一对齐到4字节。
在结构体中，综合考虑变量本身和指定的对齐值。

至于为什么输出的值是变小的，
属于另外的问题了，
简要说就是这个平台上的栈是倒着长的。
