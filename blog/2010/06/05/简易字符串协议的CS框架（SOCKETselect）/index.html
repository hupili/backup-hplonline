简易字符串协议的C/S框架（SOCKET Select）

Jun 5th, 2010

(hplonline)2010.6.5

上一篇做了服务端引擎，这篇把客户端引擎也弄好了。

下载地址：
ThunderCS2010.6.5。
SAMPLE中是一个交互式聊天的示例。

运行效果：




完成客户端编程框架。
由于引擎的主要难点在于底层的组装机制，
所以可以沿用服务器的代码。
而且客户端避免了管理多个套接字的麻烦，
从服务端代码简化即可得到。

上层代码构建方式也类似服务端，
首先派生ThunderClient类，
然后对Process和Run进行定制。

同样的，Process也会收到下层递交过来的完整PDU。

客户端向服务端发送PDU使用函数：Request。
跟服务端的函数在名字上对偶：Request/Respond。

服务端在编程的时候，一般不考虑退出的问题。
所以在最简框架中也没提供相关的机制。
如果检测到套接字失效，直接删除即可。

客户端不同，根据用户的需要应该可以主动退出。
并且客户端检测到套接字失效的时候也需要有机制通知上层。
于是提供了两个额外的函数：
Exit()，先关闭套接字，然后转换为退出状态；
Exited()，可以检测当前是否为退出状态。

于是客户端的循环框架可以变为：while ( !Exited() )

服务器端一般只处理与网络有关事件，
故在大多数情况下，可以不重载Run函数。

客户端除了处理网络事件外，
还需要处理用户的一些操作所产生的事件，
所以往往需要重载Run函数。

》》样例代码：

Proces函数直接显示收到的字符串。

重载了Run函数。
使用kbhit来判断是否按键，然后进行处理。
这样不会因为等待SOCKET或者键盘事件而阻塞了。

一般情况下，需要编写的仅为红色的部分。

class MyClient : public ThunderClient{
public:
MyClient(std::string ipAddress , unsigned short uPort , int maxcheck)
:ThunderClient(ipAddress , uPort , maxcheck){} ;

    int Process(std::string &str){
printf(“%s\n” , str.c_str()) ;
return 0 ;
}

void Run(){
std::string str = “” ;
while ( !Exited() ){
MakeReadList() ;
MakeWriteList() ;
MakeExceptList() ;
Select(0 , 10000) ;
LaunchEvent() ;
if ( kbhit() ){
char ch = getch() ;
printf(“%c” , ch) ;
if ( ch == ‘\n’ || ch == ‘\r’){
printf(“\n”) ;
Request(str) ;
if ( str == “exit” ){
Exit() ;
}
fflush(stdin) ;
str = “” ;
} else {
str += ch ;
}
}
}
}
} ;


int main(){

WORD wVersionRequested;
WSADATA wsaData;
int err;

wVersionRequested = MAKEWORD( 1, 1 );

err = WSAStartup( wVersionRequested, &wsaData);
if (err!=0){
return err;
}

if (LOBYTE ( wsaData.wVersion) != 1 ||
HIBYTE ( wsaData.wVersion) != 1){
WSACleanup();
return 0;
}

    char buf[MaxBuf] ;
puts(“input server’s ip:”) ;
